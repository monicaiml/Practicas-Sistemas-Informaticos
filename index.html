P1. Redes y gestión en Linux

En esta práctica se configura y analiza una red local entre dos máquinas Linux (cliente y servidor) en la red 192.168.50.0/24. Se realizan comprobaciones de conectividad, configuración de red y servicios.

---

1. Interfaces de red

Se usa el comando ip a para ver las interfaces de red disponibles.

ip a

Se comprueba la interfaz activa, la IP asignada y la MAC del equipo.

---

2. Configuración de red

Con ip addr y ip route se revisa la configuración de red.

ip addr
ip route

Se observa la red 192.168.50.0/24 y la interfaz usada para salir a esa red.

---

3. Hostname

Se comprueba y modifica el nombre del equipo con hostnamectl.

hostname
sudo hostnamectl set-hostname srv01

El hostname identifica la máquina dentro de la red.

---

4. Netplan

Se configura la IP estática editando el archivo de Netplan.

sudo nano /etc/netplan/01-netcfg.yaml
sudo netplan apply

Permite asignar IP fija dentro de la red local.

---

5. Ping

Se comprueba conectividad entre máquinas con ping.

ping 192.168.50.10

Se verifica que hay respuesta entre cliente y servidor.

---

6. Hosts

Se configura resolución de nombres en /etc/hosts.

sudo nano /etc/hosts
ping srv01

Permite usar nombres en lugar de IPs.

---

7. Rutas

ip route

Muestra la ruta por defecto del sistema y la red local.

---

8. Puertos

ss -tuln

Se muestran los puertos abiertos en el sistema.

---

9. SSH

sudo apt install openssh-server
sudo systemctl status ssh

Permite acceso remoto seguro al servidor.

---

10. Acceso remoto

ssh usuario@192.168.50.10
whoami
hostname

Se comprueba conexión remota al servidor.

---

11. Interfaces estado

ip link

Se comprueba si las interfaces están activas o no.

---

12. ARP

ip neigh

Se muestran direcciones IP asociadas a MAC.

---

13. SCP

scp prueba.txt usuario@192.168.50.10:/home/usuario

Se transfiere un archivo al servidor.

---

14. Gestión SSH

sudo systemctl stop ssh
sudo systemctl start ssh

Se comprueba el funcionamiento del servicio SSH.

---

15. Reinicio

Tras reiniciar se comprueba que la configuración se mantiene.

ip a
hostname
systemctl status ssh

La configuración de red y servicios se mantiene tras reinicio.
