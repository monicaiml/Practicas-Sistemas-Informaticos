P2. Redes y recursos en Windows

En esta práctica se trabaja con dos máquinas (Windows Server y Windows 11) dentro de la red 192.168.60.0/24. El objetivo es comprobar la conectividad entre equipos, configurar la red y usar recursos compartidos.

---

1. Interfaces de red

ipconfig /all

Se muestra toda la configuración de red del equipo. Aquí se puede ver la IP, la MAC y los adaptadores disponibles.

La interfaz activa es la que tiene una IP dentro de la red 192.168.60.0/24.

---

2. Adaptadores de red

Get-NetAdapter

Este comando enseña los adaptadores del sistema y su estado. El que está en “Up” es el que se está usando.

---

3. Configuración IP

ipconfig

Aquí se ve la IP asignada al equipo, la máscara de red y la puerta de enlace si existe.

La IP identifica el equipo dentro de la red local.

---

4. Tabla de rutas

route print

Muestra cómo se envían los datos por la red. Se puede ver la red local y por qué interfaz salen los paquetes.

---

5. Nombre del equipo

hostname

Muestra el nombre del equipo dentro de la red. Esto ayuda a identificar máquinas sin usar IP.

Servidor: win-srv01
Cliente: win-cli01

---

6. IP estática

New-NetIPAddress -InterfaceAlias "Ethernet" -IPAddress 192.168.60.10 -PrefixLength 24

Se asigna una IP fija al equipo dentro de la red local.

El PrefixLength 24 indica que la máscara es 255.255.255.0.

---

7. Ping

ping 192.168.60.10
ping 192.168.60.20

Sirve para comprobar si los equipos se pueden comunicar entre sí. Si responde, significa que hay conexión.

---

8. Hosts

notepad C:\Windows\System32\drivers\etc\hosts

Se usa para asignar nombres a IPs manualmente. Así se puede hacer ping por nombre en vez de IP.

---

9. Puertos

netstat -ano

Muestra conexiones activas y puertos abiertos. También indica qué programa usa cada conexión (PID).

---

10. ARP

arp -a

Relaciona IPs con direcciones MAC. Sirve para ver qué dispositivo está detrás de cada IP.

---

11. Carpeta compartida

New-SmbShare -Name red -Path C:\red

Crea una carpeta compartida para que otros equipos puedan acceder por red.

---

12. Acceso al recurso

net use Z: \\win-srv01\red

Con esto el cliente conecta la carpeta compartida como una unidad de red.

---

13. Transferencia de archivos

copy prueba.txt Z:\

Se copia un archivo al servidor a través de la unidad de red.

---

14. Conexiones activas

netstat -ano

Permite ver las conexiones que están activas en ese momento en el sistema.

---

15. Reinicio y comprobación

ipconfig
hostname
net use

Después de reiniciar se comprueba que la configuración sigue igual y que el recurso compartido sigue funcionando.
