P1. Redes y gestión en Linux

En esta práctica se trabaja con dos máquinas Linux (cliente y servidor) dentro de la red 192.168.50.0/24. El objetivo es comprobar cómo funciona la red, ver la configuración y probar la comunicación entre equipos.

---

1. Interfaces de red

ip a

Este comando muestra las interfaces de red del sistema. Aquí se puede ver la IP asignada y la dirección MAC.

La interfaz activa es la que tiene una IP dentro de la red 192.168.50.0/24.

---

2. Configuración de red

ip addr
ip route

Se revisa la configuración de red y las rutas. Esto permite ver por dónde sale la conexión a Internet o a la red local.

---

3. Hostname

hostname
sudo hostnamectl set-hostname srv01

El hostname es el nombre del equipo dentro de la red. Sirve para identificar cada máquina sin usar IP.

---

4. Netplan

sudo nano /etc/netplan/01-netcfg.yaml
sudo netplan apply

Se configura una IP fija para que el equipo no cambie de dirección cada vez que se reinicia.

---

5. Ping

ping 192.168.50.10

Se usa para comprobar si hay conexión entre los dos equipos. Si responde, la red funciona correctamente.

---

6. Hosts

sudo nano /etc/hosts
ping srv01

Permite usar nombres en vez de IPs. Es útil en redes pequeñas como esta práctica.

---

7. Rutas

ip route

Muestra cómo el sistema envía los datos por la red. Se ve la red local y la ruta por defecto.

---

8. Puertos

ss -tuln

Muestra los servicios que están escuchando en el sistema.

---

9. SSH

sudo apt install openssh-server
sudo systemctl status ssh

Se instala y comprueba SSH para poder acceder remotamente al servidor.

---

10. Acceso remoto

ssh usuario@192.168.50.10
whoami
hostname

Permite conectarse desde un equipo a otro de forma remota.

---

11. Estado de red

ip link

Muestra si las interfaces están activas (UP) o apagadas (DOWN).

---

12. ARP

ip neigh

Relaciona las IP con las direcciones MAC de los dispositivos conectados.

---

13. SCP

scp prueba.txt usuario@192.168.50.10:/home/usuario

Sirve para copiar archivos entre máquinas por SSH.

---

14. Control SSH

sudo systemctl stop ssh
sudo systemctl start ssh

Se comprueba que el servicio SSH funciona correctamente.

---

15. Reinicio

ip a
hostname
systemctl status ssh

Después de reiniciar se comprueba que la configuración sigue igual.
