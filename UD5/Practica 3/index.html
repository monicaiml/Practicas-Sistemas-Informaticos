P3. Implementación de un dominio Active Directory

En esta práctica se configura un dominio Windows con Active Directory. Un servidor actúa como controlador de dominio y un equipo Windows 11 se une a él. Esto permite gestionar usuarios y equipos de forma centralizada dentro de la red.

---

1. Configuración del servidor

Se cambia el nombre del servidor a srv-ad01 para identificarlo dentro del dominio. 

hostname

El servidor será el controlador del dominio y se encargará de gestionar toda la red.

---

2. Instalación de AD DS

Se instala el rol Active Directory Domain Services desde Server Manager.

Este rol permite crear y administrar un dominio Windows.

---

3. Creación del dominio

Se promueve el servidor como controlador de dominio creando un nuevo bosque llamado empresa.local.

Esto convierte el servidor en el centro del sistema de red.

---

4. Verificación del dominio

Se comprueba que el dominio empresa.local ha sido creado correctamente.

Aparece dentro de Active Directory Users and Computers.

---

5. Creación de usuarios

Se crean usuarios dentro del dominio: usuario1 y usuario2.

Estos usuarios ya no son locales, sino que pertenecen al dominio.

---

6. Carpeta de perfiles

Se crea la carpeta C:\perfiles en el servidor y se comparte.

Aquí se guardarán los perfiles de los usuarios del dominio.

---

7. Perfiles móviles

\\srv-ad01\perfiles\%username%

Se configura el perfil móvil para que el usuario pueda mantener su escritorio y configuración en cualquier equipo.

---

8. Unión al dominio

El equipo cliente cli01 se une al dominio empresa.local.

Esto permite que el equipo sea administrado por el servidor del dominio.

---

9. Inicio de sesión

Se inicia sesión con usuario de dominio (empresa\usuario1 y usuario2).

Se comprueba que el perfil se crea correctamente en el servidor.

---

10. GPO

Se modifica la política de contraseñas desde Group Policy Management.

Las GPO permiten controlar reglas de seguridad desde un único punto.

---

11. Aplicación de políticas

gpupdate /force

Se actualizan las políticas en el cliente para aplicar los cambios del dominio.

---

12. Comprobación final

Se verifica que:

  • El equipo está unido al dominio
  • Los usuarios inician sesión correctamente
  • Los perfiles se almacenan en el servidor
  • Las políticas de seguridad funcionan

Con esto se confirma que el dominio Active Directory funciona correctamente.
